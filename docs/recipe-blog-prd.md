# 나만의 레시피 블로그 MVP PRD

## 핵심 정보

**목적**: Notion을 CMS로 사용하여 요리 레시피를 작성하면 웹사이트에 자동 반영되어, 비개발자도 손쉽게 레시피 블로그를 운영할 수 있게 한다.
**사용자**: Notion에 익숙한 1인 요리 블로거로, 별도의 관리 도구 없이 Notion에서 직접 콘텐츠를 관리하고 싶은 사람.

---

## 사용자 여정

```
1. [홈 페이지] - 최신 레시피 카드 목록 노출
   ↓ 카테고리 / 난이도 / 조리시간 필터 선택 또는 레시피 카드 클릭

   [필터 선택] → 홈 페이지에서 결과 즉시 갱신 (페이지 이동 없음)
   [레시피 카드 클릭] → 레시피 상세 페이지 이동
   ↓

2. [레시피 상세 페이지] - 재료 / 조리 순서 / 사진 확인
   ↓ 뒤로가기 또는 홈 링크 클릭

3. [홈 페이지] - 다른 레시피 탐색 계속
```

> **콘텐츠 갱신 흐름 (블로거 관점)**
>
> Notion에 레시피 페이지 작성 → Vercel 재빌드(ISR 또는 On-Demand Revalidation) → 웹사이트 자동 반영

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|---|---|---|---|---|
| **F001** | 레시피 목록 조회 | Notion DB에서 레시피 목록을 가져와 카드 형태로 노출 | 서비스의 핵심 진입점 | 홈 페이지 |
| **F002** | 레시피 필터링 | 카테고리 / 난이도 / 조리시간으로 목록 필터 | 레시피 수 증가 시 탐색 필수 | 홈 페이지 |
| **F003** | 레시피 상세 조회 | Notion 페이지 본문을 파싱하여 재료, 조리 순서, 사진 노출 | 콘텐츠 소비의 핵심 목적 | 레시피 상세 페이지 |
| **F004** | Notion API 연동 | @notionhq/client로 DB 쿼리 및 페이지 블록 fetch | 모든 기능의 데이터 소스 | 홈 페이지, 레시피 상세 페이지 |
| **F005** | ISR / 캐시 갱신 | Next.js ISR로 일정 주기 자동 재빌드, Webhook으로 즉시 갱신 지원 | 노션 편집이 즉시 웹에 반영 | 홈 페이지, 레시피 상세 페이지 |

### 2. MVP 필수 지원 기능

## 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F002`, `F004`, `F005`, `F010`, `F011` | **인증:** 불필요

| 항목 | 내용 |
|---|---|
| **역할** | 블로그 메인 랜딩 페이지. 전체 레시피를 카드 형태로 노출하고 필터로 탐색할 수 있게 한다. |
| **진입 경로** | 사이트 접속 시 기본 진입 / 헤더 로고 또는 '레시피' 메뉴 클릭 |
| **사용자 행동** | 레시피 카드 목록 스크롤 → 카테고리 / 난이도 / 조리시간 필터 선택 → 원하는 레시피 카드 클릭 |
| **주요 기능** | 카드 그리드 (썸네일, 제목, 카테고리, 난이도, 조리시간 표시) / 카테고리 탭 필터 / 난이도 및 조리시간 드롭다운 필터 / Notion DB 실시간 데이터 fetch / **레시피 카드 클릭** 시 상세 페이지 이동 |
| **다음 이동** | 레시피 카드 클릭 → 레시피 상세 페이지 |

| **다음 이동** | 홈 링크 클릭 → 홈 페이지 |

## 데이터 모델

Notion DB가 단일 데이터 소스이므로 별도 DB는 없음. Notion DB 스키마를 아래와 같이 정의한다.

### Recipe (Notion Database 속성)

| 필드 | 설명 | 타입 / 관계 |
|---|---|---|
| title | 레시피 제목 | Title |
| category | 카테고리 | Select (한식, 양식, 일식, 중식, 기타) |
| difficulty | 난이도 | Select (쉬움, 보통, 어려움) |
| cook_time | 조리시간(분) | Number |
| servings | 인분 | Number |
| published | 공개 여부 | Checkbox |
| cover | 대표 이미지 | Files & Media |

> 레시피 본문(재료, 조리 순서)은 Notion 페이지 내부 블록(Block)으로 작성. 별도 속성 불필요.

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) - 정적 생성(SSG) + ISR로 Notion 데이터 캐싱
- **TypeScript 5.6+** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진)
- **shadcn/ui** - 카드, 드롭다운, 탭 등 컴포넌트
- **Lucide React** - 아이콘 (시계, 불꽃, 사람 아이콘 등 메타 정보 표시)

### CMS 연동

- **@notionhq/client** - 공식 Notion API 클라이언트 (DB 쿼리, 블록 fetch)
- **notion-to-md** - Notion 블록을 Markdown으로 변환 (선택적 활용)
- **react-notion-x** - Notion 페이지 렌더링 컴포넌트 (선택적 활용)

### 캐시 & 갱신

- **Next.js ISR** (`revalidate` 옵션) - 주기적 자동 재빌드
- **Next.js On-Demand Revalidation** - Notion Webhook 수신 시 즉시 갱신

### 폼 & 검증

- **Zod** - 환경변수 및 Notion API 응답 타입 검증

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포, 환경변수 관리 (NOTION_API_KEY, NOTION_DATABASE_ID)

### 패키지 관리

- **npm** - 의존성 관리
